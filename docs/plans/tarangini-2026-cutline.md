# Tarangini / 2026 Ugadi Launch Cutline

Date: March 15, 2026
Event date: May 16, 2026

Product framing for launch:
- Phone-first manager tool: Dashboard, Participants, Performances
- Tablet-first operator tool: Show Flow, Stage Console
- Supporting capability: Intro approve -> console playback, minimal act media
- Deferred: full media platform, desktop polish, broader AI/media expansion

## Must Do by March 29

- Manager path on phone must be operational:
  - Dashboard shows placement, docs, and minor-safety blockers clearly.
  - Participants supports fast search, assignment, docs follow-up, and special-request review.
  - Performances supports cast assignment, readiness review, and intro-entry access from the act workspace.
- Participant -> act assignment must be workable without desktop-only interactions.
- Waiver/doc blockers must be visible in roster and performance flows.
- Real Tarangini data must remain usable as acts and teams are finalized.
- Gate 15 blocker path must be isolated to a concrete action:
  - either reset/reseed demo validation data for rehearsal,
  - or move validation to a known-good real act with approved participant photos.

## Must Do by April 12

- Show Flow and Stage Console must be rehearsal-ready on tablet:
  - lineup reflects current act changes,
  - console shows now / next / upcoming correctly,
  - advance act flow is reliable,
  - refresh/reconnect does not break the live state.
- Gate 15 must complete its rehearsal-critical loop:
  - select approved participant photos,
  - curate,
  - generate background,
  - generate audio,
  - save,
  - approve,
  - play from Stage Console.
- Intro path must be blocked only by missing data, never by ambiguous UI or stale test IDs.
- Minimal act media handling must be good enough for music/tech records and intro prerequisites.

## Hardening Only After April 12

- Mobile and tablet touch-target cleanup on the already-shipped manager/operator path.
- Reload and reconnect resilience improvements.
- Stage Console clarity and pacing polish.
- Guardrails and empty states around partial data, missing docs, and incomplete intros.
- Performance and memory tuning only where it affects rehearsal or show-day execution.

## Deferred Until After May 16

- Gate 17 / Intro Builder overhaul.
- Rich asset library and generalized media management.
- Broader AI generation surfaces beyond current intro/background/audio MVP.
- Desktop-first polish.
- Ticketing, payments, seating, attendee/family workflows, and post-event analytics.
